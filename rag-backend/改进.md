## 详细改进方案

### 1. 🔴 Worker 日志格式不统一

**问题：** `ingest_worker.py` 用 `logging.getLogger`，其他文件用 `get_logger`，日志格式不一致。

**修改文件：** `app/workers/ingest_worker.py`

**改法：**
```python
# 改前
import logging
logger = logging.getLogger(__name__)

# 改后
from app.observability import get_logger
logger = get_logger(__name__)
```

**验证：** 启动 worker，上传文件，看终端日志是否 JSON 格式

### 2. 🔴 LLM 调用无超时

**问题：** `MiniMaxGenerator` 用 `httpx.post(timeout=120.0)`，但如果 LLM 卡住，整个请求会挂死。

**修改文件：** `app/infrastructure/generators/minimax.py`

**改法：**
```python
# 改前
response = httpx.post(..., timeout=120.0)

# 改后 - 分连接超时和读取超时
response = httpx.post(
    ...,
    timeout=httpx.Timeout(
        connect=5.0,    # 连接超时 5 秒
        read=90.0,      # 读取超时 90 秒
        write=10.0,     # 写入超时 10 秒
        pool=5.0        # 连接池超时 5 秒
    )
)
```

**验证：** 用一个不存在的 URL 测试，应该 5 秒内返回错误

### 3. 🔴 无并发限制

**问题：** 同时来 100 个请求会同时调 100 次 LLM，可能打爆 API 额度。

**修改文件：** `app/dependencies.py` + `app/routers/agent.py`

**改法：** 加一个 `threading.Semaphore` 限制并发

```python
# dependencies.py 加
import threading
_llm_semaphore = threading.Semaphore(5)  # 最多 5 个并发

def get_llm_semaphore() -> threading.Semaphore:
    return _llm_semaphore
```

```python
# agent.py 加
@router.post("/run")
def run_agent(..., semaphore = Depends(get_llm_semaphore)) -> dict:
    if not semaphore.acquire(blocking=False):
        raise HTTPException(status_code=429, detail="服务繁忙，请稍后重试")
    try:
        return rag_query_service.run(...)
    finally:
        semaphore.release()
```

**验证：** 同时发 10 个请求，应该只有 5 个成功，其他返回 429

### 4. 🔴 嵌入模型无健康检查

**问题：** 启动时不检查 embedding API 是否可用，运行时才报错。

**修改文件：** `app/dependencies.py`

**改法：** 在 `get_embedder()` 里加一个轻量探测

```python
@lru_cache
def get_embedder() -> EmbeddingProvider:
    settings = get_settings()
    embedder = build_embedder(settings)
    # 启动时探测一次
    try:
        embedder.embed_texts(["health check"])
    except Exception as e:
        logger.warning(f"Embedding API 不可用: {e}")
    return embedder
```

**验证：** 关掉 embedding 服务，重启后端，看是否有警告日志

### 5. 🟡 无请求缓存

**问题：** 相同的 prompt 每次都调 LLM，浪费钱。

**修改文件：** `app/services/rag_query_service.py`

**改法：** 加一个简单的 dict 缓存（key=prompt+collection，value=结果）

```python
from functools import lru_cache
import hashlib

def _cache_key(prompt: str, collection: str) -> str:
    return hashlib.md5(f"{prompt}:{collection}".encode()).hexdigest()

# 在 RagQueryService.__init__ 里加
self._cache: dict[str, dict] = {}
self._cache_max_size = 100
```

在 `run()` 开头检查缓存：
```python
key = _cache_key(prompt, collection)
if key in self._cache:
    logger.info("cache_hit", extra={"extra_fields": {"run_id": run_id, "cache_key": key}})
    return self._cache[key]
# ... 执行完后
if len(self._cache) >= self._cache_max_size:
    # 删除最早的缓存
    self._cache.pop(next(iter(self._cache)))
self._cache[key] = response
```

**验证：** 连续问两次相同问题，第二次应该 <100ms

### 6. 🟡 HTTP 错误码不区分

**问题：** 所有异常都返回 503，前端无法区分是参数错误还是服务挂了。

**修改文件：** `app/routers/agent.py`

**改法：**

```python
from app.errors import ValidationError, RetryableIngestionError

@router.post("/run")
def run_agent(...) -> dict:
    try:
        return rag_query_service.run(...)
    except ValidationError as exc:
        raise HTTPException(status_code=400, detail=str(exc))
    except RetryableIngestionError as exc:
        raise HTTPException(status_code=429, detail="服务繁忙，请稍后重试")
    except Exception as exc:
        detail = sanitize_error_message(str(exc))
        raise HTTPException(status_code=500, detail=f"RAG query failed: {detail}")
```

**验证：** 发一个空 prompt，应该返回 400 而不是 503

### 9. 🟢 意图分析是空实现

**问题：** `_analyze_intent` 直接返回原 query，没有真正的分析。

**修改文件：** `app/services/rag_query_service.py`

**改法（后续再做）：** 接入 LLM 做意图分类

```python
def _analyze_intent(self, prompt: str) -> dict:
    # 简单版本：关键词匹配
    keywords = ["价格", "多少钱", "退", "理赔", "保障"]
    requires_kb = any(kw in prompt for kw in keywords)
    return {
        "query": prompt,
        "requiresKnowledgeBase": requires_kb,
        "summary": "需要查询知识库" if requires_kb else "可能是闲聊",
    }
```

### 10. 🟢 无请求日志记录

**问题：** 每个 API 请求没有记录到日志，出问题无法追溯。

**修改文件：** `app/main.py`

**改法：** 加 FastAPI middleware 记录请求

```python
from starlette.middleware.base import BaseHTTPMiddleware
import time

class RequestLoggingMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        start = time.time()
        response = await call_next(request)
        elapsed = (time.time() - start) * 1000
        logger.info(
            "http_request",
            extra={"extra_fields": {
                "method": request.method,
                "path": request.url.path,
                "status": response.status_code,
                "elapsed_ms": round(elapsed),
            }}
        )
        return response

app.add_middleware(RequestLoggingMiddleware)
```

**验证：** 访问任意接口，看日志里有没有 HTTP 请求记录
